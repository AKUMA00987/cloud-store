# 日精月华 Phase 14 域名与 DNS 准备教程

这份教程只解决 3 件事：

1. 先确定正式主域名是谁
2. 在域名控制台把裸域和 `www` 都指到你的 ECS
3. 在申请证书前，先确认域名已经能通过 HTTP 正确访问到当前网站

这一步先 **不做 HTTPS**。  
你要先把域名入口跑通，再进入下一份证书教程。

## 一、这次固定用什么域名策略

当前这套教程默认按下面这个口径写：

- **主域名**：裸域
- **跳转域名**：`www`

也就是：

- 正式地址是：`https://你的域名`
- `https://www.你的域名` 最终会 `301` 跳回主域名

### 为什么这样定

- 裸域做正式主入口更直观
- 后面海报、名片、包装、用户传播时更短
- 你当前没有特别要求必须用 `www` 做主入口

如果你后面想反过来改成：

- `www` 做主域名
- 裸域跳转过去

也可以，但要整套一起改：

- DNS
- Nginx
- 验证清单

不要一半按裸域，一半按 `www`。

## 二、开始前你要准备什么

开始前，请确认：

1. 你已经知道服务器公网 IP
2. 网站已经能通过 `http://公网IP` 正常访问
3. 阿里云安全组已经放行：
   - `80`
   - `443`
   - `22`
4. 你已经能登录域名控制台

## 三、你要新增哪 2 条 DNS 记录

如果你的正式域名是：

```text
example.com
```

那你至少要准备这两条记录：

### 记录 1：裸域

- 主机记录：`@`
- 记录类型：`A`
- 记录值：你的 ECS 公网 IP

### 记录 2：`www`

- 主机记录：`www`
- 记录类型：`A`
- 记录值：同样填你的 ECS 公网 IP

## 四、域名控制台怎么操作

不同域名平台界面会不一样，但核心操作都差不多。

### 第 1 步：登录域名控制台

进入你买域名的地方，找到：

- 域名列表
- DNS 解析
- 解析设置

### 第 2 步：找到你要上线的域名

例如：

```text
example.com
```

点击进入解析管理。

### 第 3 步：新增裸域记录

新增一条：

- 主机记录：`@`
- 类型：`A`
- 记录值：你的 ECS 公网 IP

保存。

### 第 4 步：新增 `www` 记录

再新增一条：

- 主机记录：`www`
- 类型：`A`
- 记录值：同一个 ECS 公网 IP

保存。

## 五、保存后怎么判断有没有生效

DNS 不是点完立刻全球生效，通常需要一点时间。

### 方法 1：在你电脑上测试

打开浏览器，访问：

```text
http://你的域名
http://www.你的域名
```

### 方法 2：在服务器上测试

登录服务器后执行：

```bash
curl -I http://你的域名
curl -I http://www.你的域名
```

### 正常时你应该看到什么

如果当前 HTTP 已经接到你的站点，通常会看到：

- `HTTP/1.1 200 OK`

或者至少不是：

- `Could not resolve host`
- `Connection refused`
- `timeout`

## 六、如果 HTTP 访问还不通，先查哪几层

按这个顺序查：

### 1. 先看是不是 DNS 还没生效

如果浏览器提示：

- 找不到服务器
- 域名不存在

先等等，再重试。

### 2. 再看 Nginx 是否正常

在服务器执行：

```bash
sudo systemctl status nginx --no-pager
curl -I http://127.0.0.1/
```

如果这里不通，先不要继续做证书。

### 3. 再看 Node 是否正常

执行：

```bash
pm2 status
curl -I http://127.0.0.1:3000/
```

如果这里不通，也先不要继续做证书。

## 七、这一步完成时，算什么状态

完成本教程后，你应该达到这个状态：

- `http://你的域名` 能打开网站
- `http://www.你的域名` 也能打开网站
- 现在还没有启用 HTTPS
- 现在还没有强制跳转

这就是进入下一步 Certbot 的前置条件。

## 八、这一步还没有做什么

这一步 **还没有** 做：

- HTTPS 证书申请
- `443` 的 Nginx 正式配置
- HTTP 强制跳 HTTPS
- `www` 跳主域名

这些属于下一份教程：

- `root/deploy/docs/06-https-certbot-public-cutover.md`
